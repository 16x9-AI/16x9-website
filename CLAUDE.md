# 16x9 Website Builder — Conventions

**Status: mid-rebrand.** All brand-specific copy, imagery, meta tags, analytics IDs, and color values have been intentionally blanked or neutralized. This file documents structure only — no brand details should be assumed or reintroduced without explicit direction.

## Stack

- Vite + React 19 + TypeScript
- TanStack Router (file-based routing in `src/routes/`)
- TanStack Start (SSR) targeting Cloudflare Workers
- Tailwind CSS v4 (single source: `src/styles.css`)
- shadcn/ui (Radix primitives in `src/components/ui/`)
- Framer Motion available for entry/scroll animations
- Site-specific components live in `src/components/site/`

Build: `npm run build` · Dev: `npm run dev` · Preview: `npm run preview` · Lint: `npm run lint` · Format: `npm run format`

Copy voice: see `VOICE.md` at the repo root before writing or editing any user-facing copy.

## Folder structure

- `src/routes/` — file-based routes. `__root.tsx` is the root layout (head/meta, providers, error/not-found boundaries, `RootShell`). `index.tsx` is the `/` route.
- `src/routeTree.gen.ts` — auto-generated by the TanStack Router plugin on build/dev. Never hand-edit; it regenerates from the contents of `src/routes/`.
- `src/components/site/` — site-specific components (e.g. `CookieBanner`, `HeroCursorBlob`, `ScrollProgress`).
- `src/components/ui/` — shadcn/ui primitives (Radix-based). Kept as generic, reusable building blocks.
- `src/lib/analytics.ts` — analytics loader utilities (GA4, Microsoft Clarity, Identify.id pixel). IDs are currently empty strings and each loader no-ops when its ID is unset.
- `src/styles.css` — single Tailwind v4 source, theme tokens, and utility/keyframe definitions.
- `public/` — static assets served as-is.

## Token-name convention

Color tokens are defined in `src/styles.css` under `:root` and mapped into Tailwind via `@theme inline`. **Token names must be preserved** even though their values are currently neutral placeholders — other components reference them by name:

- `--ink`, `--graphite`, `--ash`, `--fog`, `--snow`, `--obsidian`, `--silver-mist`, `--slate-brand`
- shadcn/ui semantic chain: `--background`, `--foreground`, `--card`, `--popover`, `--primary`, `--secondary`, `--muted`, `--accent`, `--destructive`, `--border`, `--input`, `--ring`
- Gradient utilities: `--gradient-indigo`, `--gradient-citrus`, `--gradient-blush`
- Font variables: `--font-display`, `--font-sans` (currently a neutral system-font stack)

When rebranding, change the **values** assigned to these tokens/variables — do not rename them, since components consume them by name via Tailwind classes (`text-ink`, `bg-fog`, etc.) and CSS variables.

## Analytics & consent

`activateAnalytics()` in `src/lib/analytics.ts` loads GA4, Microsoft Clarity, and an Identify.id pixel, and is only called after the user accepts cookies in `CookieBanner`. Consent is persisted under `localStorage["cookie_consent_v1"]` (`"accepted"` | `"rejected"`). Analytics IDs are currently blank, and each loader guards on its ID being set (no-op if empty). **Never load analytics before consent** — this is a GDPR rule, not a preference.

To track a custom event from a component: `trackEvent("event_name", { event_label: "..." })`. No-op if consent was rejected or IDs are unset.

## Join the Frame form

`src/components/site/JoinForm.tsx` is the in-site application form on `/` (the "Join the Frame" section). It replaced a `mailto:info@16x9.ai` link. On submit it calls the `submitJoinApplication` server function (`src/lib/joinApplication.ts`, a TanStack Start `createServerFn`), which validates the payload server-side (required fields, email format, length caps, honeypot) and sends the answers to `info@16x9.ai` via the Resend API (`https://api.resend.com/emails`).

- Requires the `RESEND_API_KEY` server env var. See `.dev.vars.example` for local setup and the Resend account/domain-verification steps. In production it's set as a Cloudflare Worker secret: `wrangler secret put RESEND_API_KEY`.
- If the key is missing, the server function returns a `not_configured` result instead of throwing — the form still renders and validates, and the client shows "Applications are not open yet."
- The key is read via `process.env.RESEND_API_KEY` inside the server function body only; it is never imported into client code and must not appear in the client bundle.

## Routing

File-based. `/` → `routes/index.tsx` (currently a placeholder). The route tree is regenerated automatically on `npm run dev` / `npm run build` — don't hand-edit `routeTree.gen.ts`.

## Do / Don't

- **Don't** hard-code colors as hex in components. Use the Tailwind token classes (`text-ink`, `bg-fog`, etc.) so a future brand pass only needs to touch `src/styles.css`.
- **Don't** add analytics calls that fire without going through `trackEvent` / consent.
- **Don't** commit `package-lock.json` (project uses `bun.lock`).
- **Don't** add backwards-compat shims or feature flags — change the code.
- **Do** check that `npm run build` passes before committing if you've touched routing, types, or top-level deps.
